四边形不等式优化

2022-04-29

前言

不太喜欢这种优化，因介于斜率优化，wqs二分（或者凸优化）之间，

算不上一种独立的优化，只能说是其他优化的特例，

而且复杂度也不是很好，各种方面都不太行，

尤其是明明可以 O(nlog⁡n)O(n\log n)O(nlogn) 复杂度为什么要 O(n2)O(n^2)O(n2)

写起来代码倒是挺简洁的

如果会二分和单调队列（如果看这篇文章的人应该都会吧），个人建议没必要学这种优化，

直接学凸优化和斜率优化就好了

二维区间dp的优化

区间的包含与交叉

对于任意四个数 a,b,c,da,b,c,da,b,c,d（1⩽a,b,c,d⩽n1 \leqslant a,b,c,d \leqslant n1⩽a,b,c,d⩽n）

如果 a⩽b⩽c⩽da \leqslant b \leqslant c \leqslant da⩽b⩽c⩽d，则称区间 [a,d][a,d][a,d] 包含 [b,c][b,c][b,c]，区间 [a,c][a,c][a,c] 与 [b,d][b,d][b,d] 为交叉关系，

这时，对于函数 fff，如果满足 fa,c+fb,d⩽fa,d+fb,cf_{a,c}+f_{b,d} \leqslant f_{a,d}+f_{b,c}fa,c​+fb,d​⩽fa,d​+fb,c​，即交叉小于包含，那么称函数 fff 满足四边形不等式

（注意：四边形不等式也可以用于非区间dp）

四边形不等式还有一种等价的形式：

对于任意 a<ba<ba<b，fa,b+fa+1,b+1⩽fa,b+1+fa+1,bf_{a,b}+f_{a+1,b+1} \leqslant f_{a,b+1}+f_{a+1,b}fa,b​+fa+1,b+1​⩽fa,b+1​+fa+1,b​，则函数 fff 满足四边形不等式

对于函数 fff，如果对于任意两个区间 [a,d][a,d][a,d]，[b,c][b,c][b,c]，fa,d⩾fb,cf_{a,d} \geqslant f_{b,c}fa,d​⩾fb,c​，称函数 fff 对于区间包含关系具有单调性

四边形不等式定理

四边形不等式优化需要 dp 满足一些条件才能优化，就是需要满足四边形不等式

对于 二维区间dp 的经典转移方程：

dp⁡l,r=min⁡l⩽k<r{dp⁡l,k+dp⁡k+1,r+wl,r}\operatorname{dp}_{l,r}=\min\limits_{l \leqslant k < r}\{\operatorname{dp}_{l,k}+\operatorname{dp}_{k+1,r}+w_{l,r}\} dpl,r​=l⩽k<rmin​{dpl,k​+dpk+1,r​+wl,r​}

对于函数 www，如果 www 满足四边形不等式

那么 dp⁡\operatorname{dp}dp 满足四边形不等式

区间决策单调性

定义区间最优决策点函数 sss，

注：区间最优决策点函数 si,js_{i,j}si,j​ 记录的值使得

dp⁡i,j=dp⁡i,si,j+dp⁡si,j+1,j+wi,j\operatorname{dp}_{i,j}=\operatorname{dp}_{i,s_{i,j}}+\operatorname{dp}_{s_{i,j}+1,j}+w_{i,j} dpi,j​=dpi,si,j​​+dpsi,j​+1,j​+wi,j​

也就是说 si,js_{i,j}si,j​ 记录的是使得 dp⁡i,j\operatorname{dp}_{i,j}dpi,j​ 取得最优的那个断点

如果 dp⁡\operatorname{dp}dp 满足四边形不等式，那么 si,j−1⩽si,j⩽si+1,js_{i,j-1} \leqslant s_{i,j} \leqslant s_{i+1,j}si,j−1​⩽si,j​⩽si+1,j​，即 dp⁡\operatorname{dp}dp 具有决策单调性

这就意味着对于 dp⁡\operatorname{dp}dp 的转移方程：

dp⁡l,r=min⁡l⩽k<r{dp⁡l,k+dp⁡k+1,r+wl,r}\operatorname{dp}_{l,r}=\min\limits_{l \leqslant k < r}\{\operatorname{dp}_{l,k}+\operatorname{dp}_{k+1,r}+w_{l,r}\} dpl,r​=l⩽k<rmin​{dpl,k​+dpk+1,r​+wl,r​}

其中 kkk 的枚举，不需要从 lll 枚举至 r−1r-1r−1，而可以从si,j−1s_{i,j-1}si,j−1​ 枚举至 si+1,js_{i+1,j}si+1,j​

就可以优化大量复杂度

没有经过优化的算法的复杂度为 O(n3)O(n^3)O(n3)

经过优化后该算法复杂度为 O(∑l=1n−1∑r=l+1n(sl+1,r−sl,r−1+1))=O(∑l=1n−1(sl+1,n−s1,n−l+n−l))=O(n2)O(\sum\limits_{l=1}^{n-1}\sum\limits_{r=l+1}^{n}(s_{l+1,r}-s_{l,r-1}+1))=O(\sum\limits_{l=1}^{n-1}(s_{l+1,n}-s_{1,n-l}+n-l))=O(n^2)O(l=1∑n−1​r=l+1∑n​(sl+1,r​−sl,r−1​+1))=O(l=1∑n−1​(sl+1,n​−s1,n−l​+n−l))=O(n2)

四边形不等式优化二位区间dp就是这样的

例题

一维区间dp的优化

一维区间dp指的是状态转移方程如下所示的dp：

dp⁡i=min⁡0⩽j<i{dp⁡j+wj,i}\operatorname{dp}_{i}=\min\limits_{0 \leqslant j < i}\{\operatorname{dp}_{j}+w_{j,i}\} dpi​=0⩽j<imin​{dpj​+wj,i​}

对于这类方程，其优化比二维区间dp还要简单一些

首先，如果 www 满足四边形不等式，那么 dp⁡\operatorname{dp}dp 具有决策单调性

即对于 dp⁡\operatorname{dp}dp 的最优决策点函数 sss，$ s_{i-1} \leqslant s_i \leqslant s_{i+1}$

决策单调性对于一维区间dp的优化

当 dp⁡\operatorname{dp}dp 具有决策单调性时，可以将其复杂度从 O(n2)O(n^2)O(n2) 优化至 O(nlog⁡n)O(n \log n)O(nlogn)

大概是这样优化的，尝试维护一下 sss

我们用一个类似ODT的单调队列来维护它，我觉得算是维护颜色段的队列？

这个队列中的每个元素包含三个值 l,r,cl,r,cl,r,c，代表区间 [l,r][l,r][l,r] 内所有 sis_isi​ 都等于 ccc

当然，直接用ODT维护应该也行，不过时间复杂度更差

假设现在已经将队列维护完成，

那么 dp⁡\operatorname{dp}dp 也就水到渠成，

dp⁡i=dp⁡c+wc,i\operatorname{dp}_i=\operatorname{dp}_{c}+w_{c,i} dpi​=dpc​+wc,i​

O(1)O(1)O(1) 转移完毕

（注：公式的 ccc 指的是队头的元素中的 ccc）

然后用当前的 dp⁡i\operatorname{dp}_idpi​ 来更新队列

注意：接下来的 l,r,cl,r,cl,r,c 默认为队尾元素的 l,r,cl,r,cl,r,c

首先考虑，如果 iii 更新 lll 比 ccc 更新 lll 更胜一筹，那就弹出队尾元素

一直不停的弹到合格为止

但谈到合格时，也不一定合格，因为有可能对于当前区间来说，可能存在一个 k (l⩽k<r)k \ (l \leqslant k < r)k (l⩽k<r)

使得 ccc 用于更新 [l,k][l,k][l,k] 合适，iii 用于更新 [k+1,r][k+1,r][k+1,r] 合适

所以就需要将当前区间分成两部分（好像ODT中的split操作啊），那怎么确定 kkk 的值呢？

很简单，二分yyds

二分之后，就修改 rrr 为 kkk，然后，压入新元素 (k+1,n,i)(k+1,n,i)(k+1,n,i)

注意：这里为什么压入的区间为 [k+1,n][k+1,n][k+1,n] 而不是 [k+1,r][k+1,r][k+1,r]，原因是要时刻保证队列能够覆盖整个区间 [1,n][1,n][1,n]

稍微算一下时间复杂度，首先外层循环为 O(n)O(n)O(n)，里层有一个二分和一个循环弹出，二分是 O(log⁡n)O(\log n)O(logn)，但循环弹出是多少呢？可以发现，循环弹出在外层循环的过程中，合计 O(n)O(n)O(n)，所以其时间复杂度为 O(nlog⁡n)O(n \log n)O(nlogn)

注意：这里介绍的其实是单调队列+wqs二分的解法，也可以被称为凸优化，还可以被称为斜率优化，总之名字很多就对了

例题

这里的例题是 NOI2009的《诗人小G》

这道题可以光速列出转移方程和状态设计

令 dp⁡i\operatorname{dp}_idpi​ 表示前 iii 个句子排版后最小的不协调度

状态转移方程如下：（L，PL，PL，P 与题目描述一样，aia_iai​ 表示第 iii 个句子的长度）

dp⁡i=min⁡0⩽j<i{dp⁡j+∣∑k=j+1iak−L−1∣P}\operatorname{dp}_{i}=\min\limits_{0 \leqslant j < i}\{\operatorname{dp}_{j}+\lvert \sum\limits_{k=j+1}^ia_k-L-1 \rvert^P \} dpi​=0⩽j<imin​{dpj​+∣k=j+1∑i​ak​−L−1∣P}

然后……O(n3P)O(n^3P)O(n3P) 的时间复杂度，就只能过第一个点，拿到 101010 分

接下来开始优化，首先是前缀和， ∑k=j+1iak\sum\limits_{k=j+1}^ia_kk=j+1∑i​ak​ 显然可以用前缀和优化到 O(1)O(1)O(1)

现在时间复杂度变为 O(n2P)O(n^2P)O(n2P)，能拿 303030 分

PPP 次方可以用快速幂算，优化成 O(n2log⁡P)O(n^2 \log P)O(n2logP)，不过并不能带来任何分数上的改变

接下来就是四边形不等式了，因为 ∑k=j+1iak\sum\limits_{k=j+1}^ia_kk=j+1∑i​ak​ 是符合四边形不等式的，

而 −L−1-L-1−L−1 以及 PPP 都是常数，所以 ∣∑k=j+1iak−L−1∣P\lvert \sum\limits_{k=j+1}^ia_k-L-1 \rvert^P∣k=j+1∑i​ak​−L−1∣P 是符合四边形不等式的

然后就可以根据决策单调性，优化状态转移为 O(nlog⁡n)O(n \log n)O(nlogn)，算上计算 PPP 次方的时间，

总共 O(nlog⁡nlog⁡P)O(n \log n\log P)O(nlognlogP)

但这道题最麻烦的地方在于需要输出一组可行解，所以最后需要记录一下转移路径

详情请看代码：

#include<iostream>
#include<cstdio>
#include<cstring>
using namespace std;
#define ld long double
#define ll long long
const int MAXN=1e5+7;
int T,n,l,p;
struct Node
{
    int l,r,c;
    Node():l(0),r(0),c(0) {}
    Node(int l,int r,int c):l(l),r(r),c(c) {}
} q[MAXN];
int hd,tl;
ld pre[MAXN],dp[MAXN];
int lst[MAXN],ans[MAXN],nxt[MAXN];
char s[MAXN][33];

ld qpow(ld a,int b)
{
    ld ret=1;
    while(b)
    {
        if(b&1) ret*=a;
        b>>=1;
        a*=a;
    }
    return ret;
}

ld trans(int from,int to)
{
    return dp[from]+qpow(abs(pre[to]-pre[from]-l-1),p);
}

inline void read(int &x)
{
    char ch;
    bool flag=0;
    x=0;
    while(ch=getchar(),!isdigit(ch))if(ch=='-')flag=1;
    while(isdigit(ch))x=x*10+ch-'0',ch=getchar();
    if(flag)x=-x;
}


int main()
{
    read(T);
    while(T--)
    {
        memset(q,0,sizeof q);
        read(n),read(l),read(p);
        for(int i=1; i<=n; i++)
        {
            scanf("%s",s[i]);
            pre[i]=pre[i-1]+strlen(s[i])+1;
        }
        q[hd=tl=1]=Node(1,n,0);
        for(int i=1; i<=n; i++)
        {
            while(hd<tl&&q[hd].r<i) ++hd;
            ++q[hd].l;
            dp[i]=trans(q[hd].c,i);
            lst[i]=q[hd].c;
            while(hd<tl&&trans(i,q[tl].l)<=trans(q[tl].c,q[tl].l)) --tl;
            int ls=q[tl].l,rs=q[tl].r;
            int now=q[tl].c,k=rs+1;
            while(ls<=rs)
            {
                int mid=(ls+rs)>>1;
                if(trans(i,mid)<=trans(now,mid)) rs=(k=mid)-1;
                else ls=mid+1;
            }
            if(k!=q[tl].l) q[tl].r=k-1;
            else --tl;
            if(k<=n) q[++tl]=Node(k,n,i);
        }
        if(dp[n]>1e18) puts("Too hard to arrange");
        else
        {
            printf("%lld\n",(ll)(dp[n]+0.5));
            for(int i=n; i; i=lst[i]) nxt[lst[i]]=i;
            int now=0;
            for(int i=1; i<=n; i++)
            {
                now=nxt[now];
                for(int j=i; j<now; j++) printf("%s ",s[j]);
                printf("%s\n",s[now]);
                i=now;
            }
        }
        puts("--------------------");
    }
    return 0;
}

四边形不等式优化整体思路

讲了两种模板型dp的四边形不等式优化，大概也能找到四边形不等式优化的精髓了

首先明确四边形不等式的目的是证明决策单调性

所以对于任意一个问题的四边形不等式优化，分为三步：

  1. 证明 www 函数满足四边形不等式
  2. 证明 dp⁡\operatorname{dp}dp 满足四边形不等式
  3. 证明决策单调性

最后用决策单调性优化

这里决策单调性可以用二分优化到 O(nlog⁡n)O(n\log n)O(nlogn)，也可就直接暴力 O(n2)O(n^2)O(n2)

四边形不等式的本质是证明函数为凸，然后以此证明决策单调性，

所以可以对其wqs二分

同时，四边形不等式优化也是斜率优化的一种

最后的例题

这里挑选的例题是CF321E

秒速想出状态转移方程和状态设计（毕竟有些难度，光速是不大可能了）

dp⁡i,j\operatorname{dp}_{i,j}dpi,j​ 表示前 i 个人，乘坐了 j 条船，所产生的最小沮丧值

令 wi,jw_{i,j}wi,j​ 表示从第 i 个人到第 j 个人如果坐在同一条船上，会产生的沮丧值

转移方程如下：

dp⁡l,r=min⁡l⩽k<r{dp⁡l,k+wk,r}\operatorname{dp}_{l,r}=\min\limits_{l \leqslant k < r}\{\operatorname{dp}_{l,k}+w_{k,r} \} dpl,r​=l⩽k<rmin​{dpl,k​+wk,r​}

www 满足四边形不等式，可证明 dp⁡\operatorname{dp}dp 也满足

所以 dp⁡\operatorname{dp}dp 满足区间单调性

然后这道题就完成了

是wqs二分还是暴力对于这道题都可以，毕竟数据范围很小
